## Codebase Patterns
- Design tokens are CSS custom properties in src/ui/styles.css, organized by theme (:root = dark, [data-theme="light"], [data-theme="high-contrast"])
- Reference source of truth for all design tokens: httptoolkit-ui/src/styles.ts
- Box shadows use `rgb(0 0 0 / var(--box-shadow-alpha))` so they auto-adapt to themes (dark=0.4, light=0.3, high-contrast=0.1)
- Font size tokens follow naming: `--font-size-{small-print|text-input|text|sub-heading|heading|large-heading|loud-heading|screaming-heading}`
- Link colors use `--link-color` and `--visited-link-color` variables (theme-aware)
- The project is vanilla JS (no framework, no TypeScript) — "typecheck" means verifying `node --check` passes on JS files
- All border-radius=4px, resizer=11px, sidebar=75px, header/footer=38px, row=32px
- Phosphor Icons loaded via `<link rel="stylesheet" href="/vendor/phosphor/regular/style.css">` — served from node_modules via Express static at `/vendor/phosphor`
- Phosphor icon class format: `<i class="ph ph-icon-name"></i>` (regular weight), size controlled via `font-size` CSS
- Sidebar icons: Intercept=ph-plug, View=ph-magnifying-glass, Mock=ph-pencil-simple, Send=ph-paper-plane-tilt, Settings=ph-gear
- All action/button icons use Phosphor `<i class="ph ph-icon-name">` elements (not inline SVGs). Only interceptor card brand icons (Chrome, Firefox, etc.) and decorative empty-state illustrations remain as SVGs.
- Traffic source icons: Globe, Terminal, Cube (Docker), GearSix (mock), FolderOpen (import), LockSimpleOpen (tls-error)
- CSS `.source-proxy i.ph`, `.source-mock i.ph`, `.source-import i.ph` set per-source icon colors
- Mock rule action Phosphor icons: FloppyDisk (save), PencilSimple (edit), ToggleRight/ToggleLeft (enable/disable), Tag (rename), CopySimple (clone), TrashSimple (delete)
- Pin icon uses PushPin with `transform:rotate(45deg)` when unpinned, `transform:none` when pinned
- For dynamic icon switching (e.g. togglePause), update `btn.innerHTML` with Phosphor `<i>` elements
- Monaco Editor served via Express static at `/vendor/monaco` from `node_modules/monaco-editor/min`
- Monaco loaded via AMD loader: `<script src="/vendor/monaco/vs/loader.js">` + `require.config({ paths: { vs: '/vendor/monaco/vs' } })`
- Use `createMonacoEditor(containerId, options)` to create editors — returns a Promise<editor>
- Monaco custom themes: `httptoolkit-dark` and `httptoolkit-light` — auto-synced via `setTheme()`
- `monacoReady` promise resolves when Monaco API is available; `monacoApi` holds the global reference
- `monacoInstances` array tracks all active editors for theme switching and cleanup
- Editor auto-resizes via ResizeObserver; auto-cleans up when container is removed from DOM
- Body viewers use dual-container pattern: `id="*-monaco"` (Monaco) + `id="*-fallback"` (pre); toggled by `isMonacoViewMode(mode)` — used by both request body (`reqBody-monaco`) and response body (`resBody-monaco`)
- `activeBodyEditors` tracks live Monaco body editors by container id; call `disposeBodyEditor(id)` before DOM replacement
- Non-Monaco body view modes: hex (textToHex), decoded (key-value grid), image (placeholder) — all render in the fallback `<pre>`
- Send page body editor: `sendBodyEditor` variable holds the editable Monaco instance; use `getSendBodyValue()` / `setSendBodyValue()` to read/write; `updateSendBodyLanguage()` syncs language from format dropdown
- `monacoApi.editor.setModelLanguage(model, lang)` switches language without recreating the editor instance
